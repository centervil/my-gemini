# Skill: GitHub Issue Manager

GitHub上へのIssue登録を、プロジェクトの統一基準（ラベル、テンプレート）に則って自動化します。

## Description
ユーザーからバグ報告や機能要望があった際、適切なテンプレートを選択し、統一されたラベルを付与して `gh issue create` を実行します。

## Usage Guidelines



1. **タイプ判別と下書き作成**:

   ユーザーの入力から `bug`, `feature/enhancement`, `question`, `reflect` のいずれかに分類し、テンプレート（`scripts/get-issue-template.sh`）とラベルガイドライン（`references/label-guidelines.md`）を使用してIssueの下書き（タイトル、ラベル、本文）を作成します。



2. **ユーザーへの確認（必須）**:

   作成した下書きをユーザーに提示し、内容（特にタイトルとラベルが意図通りか）の確認を得てください。**この際、ユーザーの入力から読み取れない不明確な情報がある場合は、勝手に推測して埋めるのではなく、必ずユーザーに質問して情報を補完してください。**



3. **Issueの作成**:

   ユーザーの承認が得られた後に、以下のコマンドを使用してIssueを作成します：

   ```bash

   gh issue create --title "[タイトル]" --body "[テンプレートを埋めた本文]" --label "[選定したラベル]"

   ```



## 注意事項

- **推測の禁止**: ユーザーが提供していない詳細情報（再現手順や具体的な期待値など）をAIが勝手に創作してはいけません。不足している場合は「不明」とするか、ユーザーに問いかけてください。

- **最終承認**: `gh issue create` を実行する前に、必ずユーザーから「作成してOK」という旨の明示的な合意を得てください。

- **既存Issueの確認**: `gh issue list` を使用して、似た内容のIssueが既に存在しないか確認することを推奨します。
